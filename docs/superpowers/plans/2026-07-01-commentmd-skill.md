# commentmd Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Qoder CLI skill `/commentmd <path>` that opens a browser tab, lets the user highlight-and-comment a Markdown file, and returns a structured JSON of comments to the Agent.

**Architecture:** A single Python stdlib script (`serve.py`) starts a local HTTP server, injects the markdown into a self-contained `viewer.html`, opens the browser, and blocks until the user submits. Submission writes `<md_name>.comments.json` next to the source and shuts the server down. The frontend uses `marked` (CDN) for rendering and DOM `Range` APIs to record TextQuoteSelector-style anchors (`quote` + `prefix` + `suffix`).

**Tech Stack:** Python 3 (stdlib only: `http.server`, `webbrowser`, `hashlib`, `json`, `argparse`, `pathlib`, `socket`), vanilla HTML/CSS/JS in one file, `marked` via CDN, `unittest` for Python-side tests.

## Global Constraints

- Runtime dependencies: **Python stdlib only** (spec §6.2). No `pip install`.
- Default port: **3118**, auto-scan up to **3128** for a free port (spec §7).
- Host: bind to **127.0.0.1** only, never `0.0.0.0` (local-only tool).
- Output default: `<md_path_without_ext>.comments.json` in the same directory as the source md.
- JSON schema version constant: **`"schema_version": 1`** (spec §6.4).
- Comment id format: sequential `c1`, `c2`, ... generated by the frontend in creation order.
- Prefix/suffix window: **32 characters** on each side of the selection (spec §6.3).
- Frontend fetches `marked` from `https://cdn.jsdelivr.net/npm/marked/marked.min.js`.
- Skill install path (final): `~/.agents/skills/commentmd/` (symlink to the dev repo during development).
- Dev repo root: `/Users/rehe/work/commentmd/`.
- Tests run from repo root via `python -m unittest discover -s tests -v`.

import socket


def find_free_port(start: int, end: int) -> int:
    for port in range(start, end + 1):
        with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as sock:
            try:
                sock.bind(("127.0.0.1", port))
            except OSError:
                continue
            return port
    raise RuntimeError(f"No free port in [{start}, {end}]")
```

import json


def build_page_data(md_path: Path) -> dict:
    md_path = md_path.resolve()
    content = md_path.read_text(encoding="utf-8")
    return {
        "md_file": str(md_path),
        "md_name": md_path.name,
        "md_content": content,
        "md_sha256": compute_sha256(md_path),
    }


def resolve_output_path(md_path: Path, out_arg):
    if out_arg is not None:
        return out_arg
    return md_path.with_suffix(".comments.json")


_SENTINEL = "/* __DATA_INJECTION__ */"


def inject_template(html_template: str, data: dict, mode: str) -> str:
    # json.dumps escapes </ so a literal </script> can't leak into the JS
    payload = json.dumps(data, ensure_ascii=False).replace("</", "<\\/")
    mode_js = json.dumps(mode)
    block = f'window.__DATA__ = {payload};\nwindow.__MODE__ = {mode_js};'
    if _SENTINEL in html_template:
        return html_template.replace(_SENTINEL, block)
    injection = f'<script>\n{block}\n</script>\n</head>'
    return html_template.replace("</head>", injection, 1)
```

    function findQuoteRange(quote, prefix, suffix) {
      const app = document.getElementById("app");
      const full = app.innerText;
      let idx = -1;
      if (prefix) {
        const anchor = full.indexOf(prefix + quote);
        if (anchor >= 0) idx = anchor + prefix.length;

      if (idx < 0) return null;
      // Walk text nodes to convert (idx, quote.length) into a Range.
      const walker = document.createTreeWalker(app, NodeFilter.SHOW_TEXT);
      let cursor = 0;
      let startNode = null, startOff = 0, endNode = null, endOff = 0;

# commentmd

## 触发时机

- 用户显式输入 `/commentmd <md_path>`。
- Agent 生成了一份 Markdown（技术方案、设计文档、README 等）并需要人类审阅意见时，主动建议使用本 skill。

## 用法

```
/commentmd <md_path>
```

参数：`<md_path>` 为待审阅 Markdown 文件的相对或绝对路径。

## Agent 执行步骤

1. 解析 `<md_path>`，转成绝对路径 `$ABS`。
2. 运行：
   ```bash
   python ~/.agents/skills/commentmd/scripts/serve.py "$ABS"
   ```
3. 命令会启动本地 HTTP server（127.0.0.1:3118 或下一个空闲端口），自动打开浏览器。用户在浏览器里划词评论并点击「完成评论」后，命令返回 `wrote <out_path>` 并退出。
4. 读取 `<out_path>`（默认与 md 同目录，文件名 `<md名去后缀>.comments.json`）。
5. 逐条阅读评论：`quote` 是被引用的原文片段，`prefix` / `suffix` 是前后 32 字符窗口，`comment` 是用户意见。若 `md_changed_during_review` 为 `true`，提醒用户原文已在审阅期间被外部修改。
6. 根据评论修订原文，并在回复中总结如何回应每条意见。

## 无浏览器 / 远程环境

追加 `--static /tmp/review.html`：
```bash
python ~/.agents/skills/commentmd/scripts/serve.py "$ABS" --static /tmp/review.html
```
用户在自己的机器上打开这个 HTML，点击「完成评论」会下载一份 JSON。Agent 需要用户告知下载文件路径后读取。

## 输出 JSON 格式

```json
{
  "schema_version": 1,
  "md_file": "/abs/path/技术方案.md",
  "md_sha256": "abc...",
  "md_changed_during_review": false,
  "created_at": "2026-07-01T10:00:00Z",
  "comment_count": 2,
  "comments": [
    {
      "id": "c1",
      "quote": "使用 MySQL 存储事件",
      "prefix": "存储层设计中，我们",
      "suffix": "，通过主从复制",
      "comment": "为什么不用 PostgreSQL？JSONB 支持更好",
      "created_at": "2026-07-01T10:00:12Z"
    }
  ]
}
```

`comment_count: 0` 表示用户无异议。
